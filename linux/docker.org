* docker常用命令
** 帮助名令

     docker version # 查看版本信息

** 镜像命令
*** docker images 查看本地镜像信息
    可选参数
    #+begin_src shell 
      Usage:  docker images [OPTIONS] [REPOSITORY[:TAG]]

      List images

      Options:
	-a, --all             Show all images (default hides intermediate images) # 列出所有镜像
	    --digests         Show digests
	-f, --filter filter   Filter output based on conditions provided # 过滤
	    --format string   Pretty-print images using a Go template
	    --no-trunc        Don't truncate output
	-q, --quiet           Only show image IDs  # 只显示ID
    #+end_src
*** docker search 搜索镜像
    可选参数
    #+begin_src shell
      Usage:  docker search [OPTIONS] TERM

      Search the Docker Hub for images

      Options:
	-f, --filter filter   Filter output based on conditions provided # 过滤
	    --format string   Pretty-print search using a Go template
	    --limit int       Max number of search results (default 25)
	    --no-trunc        Don't truncate output
    #+end_src
    例如： docker search mysql -f STARS=3000 搜索收藏大于3000以上的镜像
*** doker pull 下载镜像
    可选参数
    #+begin_src shell
      Usage:  docker pull [OPTIONS] NAME[:TAG|@DIGEST]

      Pull an image or a repository from a registry

*** docker rmi 删除镜像
    rmi＝rm remove i images
    可选参数
    #+begin_src shell
      Usage:  docker rmi [OPTIONS] IMAGE [IMAGE...]

      Remove one or more images

      Options:
	-f, --force      Force removal of the image # 强制删除
	    --no-prune   Do not delete untagged parents
    #+end_src

    例子
    #+begin_src shell
      docker rmi -f id # 删除指定镜像
      docker rmi -f id id id id # 删除指定的多个镜像
      docker rmi -f $(docker images -qa) # 递归删除所有镜像
    #+end_src
*** docker commit镜像
    docker commit 提交容器成为一个新的副本

    命令：
    #+begin_src shell
      docker commit -m='描述' -a='作者' id targetName:tag
    #+end_src

    可选参数：
    #+begin_src shell
      Usage:  docker commit [OPTIONS] CONTAINER [REPOSITORY[:TAG]]

      Create a new image from a container's changes

      Options:
	-a, --author string    Author (e.g., "John Hannibal Smith <hannibal@a-team.com>")
	-c, --change list      Apply Dockerfile instruction to the created image
	-m, --message string   Commit message
	-p, --pause            Pause container during commit (default true)
    #+end_src

    测试：

    启动一个默认的tomcat，基本的镜像webapps目录下没有内容，将webapps.dist目录下的内容拷贝到webapps下面，commit镜像。

    #+begin_src shell
      docker commit -a='wuguipeng' -m='test' d76616f702d2 tomcat-test:1.0
    #+end_src
    执行之后就可以在docker images下面看到这个新打包的镜像了，后续可以直接使用这个镜像，也可以将这个镜像发布给到镜像仓库。

** docker ps 查看运行中的镜像
   可选参数
   #+begin_src shell 
     Usage:  docker ps [OPTIONS]

     List containers

     Options:
       -a, --all             Show all containers (default shows just running) # 查看运行历史
       -f, --filter filter   Filter output based on conditions provided # 过滤
	   --format string   Pretty-print containers using a Go template
       -n, --last int        Show n last created containers (includes all states) (default -1)
       -l, --latest          Show the latest created container (includes all states)
	   --no-trunc        Don't truncate output
       -q, --quiet           Only display container IDs # 查看运行的id
       -s, --size            Display total file sizes # 查看运行的文件大小
   #+end_src

   例子：
   #+begin_src shell
     docker ps # 查看运行中的镜像
     docker ps -a # 查看当前运行的容器和历史记录
     docker ps -n=1 #查看最近一条历史
     docker ps -l # 查看最近历史
   #+end_src
   
** exit退出容器
    #+begin_src shell
      exit # 直接退出容器并停止容器

** docker rm删除容器
    #+begin_src shell
      docker rm id # 删除容器，不能删除正在运行的容器
      docker rm -f id #强制删除容器
      docker rm -f $(docker ps -aq) # 强制删除所有容器
      docker ps -a -q|xargs docker rm # 强制删除所有容器
    #+end_src
** docker start|top|restart|kill启动和停止容器
    #+begin_src shell
      docker start id # 启动一个容器
      docker restart id # 重启一个容器
      docker stop id #停止一个正在运行的容器
      docker kill id #强制停止一个容器
    #+end_src

** 什么是镜像
   镜像是一种轻量级、可执行的独立软件包，用来打包软件运行环境和基于运行环境开发的软件，它包含运行某个软件所需要的所有内容，包括代码、运行时、库、环境变量和配置文件等等。

   如何获得镜像：
   + 从远程开源仓库下载
   + 自己制作

** Docker镜像加载原理
   /UnionFS（联合文件系统）/

   UnionFS（联合文件系统）：Union文件系统（UnionFS）是一种分层、轻量级并且高性能的文件系统，它支持对文件系统的修改作为一次提交来一层一层的叠加，同时可以将不同目录挂载到同一个虚拟文件系统下。Union文件系统是Docker镜像的基础。镜像可以通过分层来继承，基于基础镜像（没有父镜像），可以制作各种具体的应用镜像。

   特性：一次同时加载多个文件系统，但从外面看起来，只能看到一个文件系统，联合加载会把各层系统叠加起来，这样最终的文件系统会包含所有底层的文件和目录。

** DockerFile 构建过程
   /基础知识：/
   + 每个保留关键字(指令)都必须是大写字母
   + 执行从上到下顺序执行
   + # 表示注释
   + 每一个指令都会创建提交一个新得 *镜像层*, 并提交！

   dockerfile是面向开发得，发布项目，做镜像，就需要编写dockerfile文件，这个文件不难。

   Dockerfile： 构建文件，定义了构建步骤

   DockerImages： 通过Dockerfile构建生成镜像，最终发布和运行产品

   Docker容器： 容器是用来运行镜像的
   

   
** Dockerfile指令
   | 指令       | 作用                                             |
   |------------+--------------------------------------------------|
   | FROM       | 指定基础镜像                                     |
   | MAINTAINER | 维护者，姓名+邮箱                                |
   | RUN        | 运行                                             |
   | ADD        | 添加其他镜像                                     |
   | WORKDIR    | 工作目录                                         |
   | VOLUME     | 设置卷，挂载目录                                 |
   | EXPOSE     | 指定对外的端口                                   |
   | CMD        | 指定容器启动的时候运行的命令，只有最后一个会生效 |
   | ENTRYPOINT | 指定容器启动的时候运行的命令，可以追加命令       |
   | ONBUILD    | 构建一个被集成 Dockerfile,就会运行               |
   | COPY       | 类似ADD,将文件拷贝到镜像中                       |
   | ENV        | 构建的时候设置环境变量                           |
   |------------+--------------------------------------------------|


   *例子：*
   #+begin_src shell
     # docker hub 中的大部分镜像都是从这个scratch开始的，是最基础的镜像

     FROM centos
     MAINTAINER wuguipeng<rskdwapy@gmail.com>

     ENV MYPATH /usr/local
     WORKDIR $MYPATH

     RUN yum -y install vim
     RUN yum -y install net-tools

     EXPOSE 80

     CMD echo $MYPATH
     CMD echo "===================end======================"
     CMD /bin/bash


     # 编写完dockerfile之后，使用docker build -f dockerfile -t mycentos:1.0 .
     # -f 是构建文件dockerfile 文件中必须包含上面的内容，
     # -t 是目标镜像:tag，最后还有一个点，代表当前目录
   #+end_src

** CMD 和ENTRYPOINT的区别
   在dockerfile文件中使用CMD指令进行构建之后，docker run运行时可以使用额外的命令替换dockerfile文件中的命令。

   在dockerfile文件中使用ENTRYPOINT指令进行构建之后，docker run运行时使用额外的命令不会替换dockerfile中的命令，而是在dockerfile中的命令后面进行追加。

** redis集群
   redis需要创建6个服务，3台主机+3台从机
   
   /通过脚本创建6个reids配置问价：/
   #+begin_src shell
     for port in $(seq 1 6);\
     do \
     mkdir -p /mydata/redis/node-${port}/conf
     touch /mydata/redis/node-${port}/conf/redis.conf
     cat << EOF >/mydata/redis/node-${port}/conf/redis.conf
     port 6379
     bind 0.0.0.0
     cluster-enabled yes
     cluster-config-file nodes.conf
     cluster-node-timeout 5000
     cluster-announce-ip 172.38.0.1${port}
     cluster-announce-port 6379
     cluster-announce-bus-port 16379
     appendonly yes
     EOF
     done
     # 启动6个redis容器
     docker run -p 637${port}:6379 -p 1637${port}:16379 --name redis-${port} \
     -v /mydata/redis/node-${port}/data:/data \
     -v /mydata/redis/node-${port}/conf/redis.conf:/etc/redis/redis.conf\
     -d --net redis --ip 172.38.0.1${port} redis redis-server  /etc/redis/redis.conf;
   #+end_src

   /创建集群：/
   #+begin_src shell
     redis-cli --cluster create 172.38.0.11:6379 172.38.0.12:6379 172.38.0.13:6379 172.38.0.14:6379 172.38.0.15:6379 172.38.0.16:6379 --cluster-replicas 1
   #+end_src
